# UVa 10359 - Tiling

## 解題思路

這題要計算有多少種方法可以鋪滿一個 **2 × n** 的長方形。

可以使用兩種磚塊：

- **2 × 1** 直立放
- **2 × 2** 正方形（也可視為兩塊橫放的 2 × 1）

題目要求：

> **計算共有多少種不同的鋪法。**

由於答案會非常大，因此必須使用 **Big Integer（大數）**。

這題是一道典型的 **Dynamic Programming（DP）**。

---

## 關鍵邏輯

### 1. DP 定義

令：

```cpp
dp[i]
```

表示：

```text
鋪滿 2 × i 的方法數。
```

---

### 2. 狀態轉移

最後一塊有兩種可能：

#### (1) 放一塊直立磚

占用：

```text
2 × 1
```

前面剩：

```text
2 × (i-1)
```

因此：

```cpp
dp[i-1]
```

種方法。

---

#### (2) 放兩塊橫放磚（或一塊 2×2）

占用：

```text
2 × 2
```

前面剩：

```text
2 × (i-2)
```

但有兩種擺法：

- 一塊 2×2
- 兩塊橫放的 2×1

因此共有：

```cpp
2 × dp[i-2]
```

種方法。

---

所以：

```cpp
dp[i] = dp[i-1] + 2 × dp[i-2];
```

---

### 3. 初始條件

當：

```text
n = 0
```

代表不用放任何磚：

```cpp
dp[0] = 1;
```

當：

```text
n = 1
```

只能放一塊直立磚：

```cpp
dp[1] = 1;
```

---

## 範例推導

假設：

```text
n = 4
```

DP：

```text
dp[0] = 1

dp[1] = 1

dp[2] = 3

dp[3] = 5

dp[4] = 11
```

因此答案：

```text
11
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 遞迴式不是 Fibonacci。
2. 應為：

```cpp
dp[i] = dp[i-1] + 2 * dp[i-2];
```

3. 答案非常大，不能使用 `long long`。
4. 需要使用 Big Integer。
5. 題目直到 EOF。

---

## 總結（Takeaway）

這題是 Big Integer DP 的經典題。

解題流程如下：

1. 預先計算所有 `dp`。
2. 使用：

```cpp
dp[i] = dp[i-1] + 2 * dp[i-2];
```

3. 使用大數儲存答案。
4. 每次查詢直接輸出。

**Time Complexity：** `O(n × L)`

- `n`：最大長度（250）
- `L`：大數位數

**Space Complexity：** `O(n × L)`
