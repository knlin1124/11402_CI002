# UVa 11034 - Ferry Loading IV

## 解題思路

這題要模擬渡輪載車。

渡輪一次只能停靠：

```text
左岸（left）
或
右岸（right）
```

每台車都有：

- 車長（公分）
- 所在岸邊（left / right）

渡輪長度是：

```text
L (公尺)
```

因此先換算成：

```text
L × 100 (公分)
```

每次渡輪會盡可能把目前岸邊可以載的車全部載上，再開到另一岸。

最後要求：

> **最少需要航行幾次。**

由於車子都是依照到達順序排隊，因此最適合使用 **Queue（佇列）**。

---

## 關鍵邏輯

### 1. 建立兩個 Queue

分別儲存：

```text
Left Queue

Right Queue
```

依照輸入順序放入。

例如：

```text
Left：

250
300
400
```

```text
Right：

150
200
```

---

### 2. 每次盡量載滿

假設目前在：

```text
Left
```

剩餘容量：

```text
capacity = L
```

依序查看：

```cpp
queue.front()
```

若放得下：

```cpp
capacity -= length;
queue.pop();
```

一直載到：

```text
下一台放不下
```

或：

```text
Queue 空了
```

---

### 3. 開到另一岸

載完後：

```text
Left → Right

Right → Left
```

並且：

```cpp
trip++;
```

直到：

```text
兩個 Queue 都空
```

---

## 範例推導

假設：

```text
L = 5 公尺
```

容量：

```text
500 公分
```

左岸：

```text
250

250

300
```

右岸：

```text
100
```

第一次：

```text
Left

250 + 250 = 500
```

載滿。

航行：

```text
1
```

第二次：

```text
Right

100
```

航行：

```text
2
```

第三次：

```text
Left

300
```

航行：

```text
3
```

答案：

```text
3
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 一定要先把公尺換成公分。
2. 左右岸都需要一個 Queue。
3. 每次都要盡可能載滿。
4. 即使沒有載到車，只要需要到另一岸，也算一次航行。
5. 題目有多組測資。

---

## 總結（Takeaway）

這題是 Queue 模擬的經典題。

解題流程如下：

1. 建立左右兩個 Queue。
2. 每次盡可能載滿目前岸邊的車。
3. 開往另一岸。
4. 重複直到兩岸都沒有車。
5. 輸出總航行次數。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
