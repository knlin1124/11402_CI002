# UVa 10931 - Parity

## 解題思路

這題要把輸入的十進位整數轉成二進位，並計算二進位表示法中有幾個 `1`。

這個 `1` 的數量就是題目所說的 **Parity**。

題目會一直輸入整數，直到輸入 `0` 為止。

---

## 關鍵邏輯

### 1. 十進位轉二進位

可以不斷對數字取餘數：

```cpp
n % 2
```

取得目前最低位元。

接著把數字除以 2：

```cpp
n /= 2
```

直到 `n` 變成 0。

---

### 2. 計算 1 的數量

每次取出來的位元如果是 `1`，就讓計數器加一。

```cpp
if (n % 2 == 1) {
    count++;
}
```

---

### 3. 注意二進位順序

因為用 `% 2` 取出的位元是從右到左，所以存成字串時會是反的。

最後要反轉字串，才會得到正確的二進位表示。

---

## 範例推導

假設輸入：

```text
7
```

轉成二進位：

```text
111
```

其中有三個 `1`，所以輸出：

```text
The parity of 111 is 3 (mod 2).
```

假設輸入：

```text
10
```

轉成二進位：

```text
1010
```

其中有兩個 `1`，所以輸出：

```text
The parity of 1010 is 2 (mod 2).
```

---

## 語言差異與踩坑紀錄

這題容易錯的地方：

1. 輸入 `0` 時不要輸出。
2. 二進位字串要反轉。
3. 輸出格式要完全一樣：

```text
The parity of B is C (mod 2).
```

其中：

- `B` 是二進位表示法。
- `C` 是 `1` 的數量。

---

## 總結（Takeaway）

這題是基本的進位轉換題。

解題步驟如下：

1. 持續讀入整數。
2. 若輸入為 `0`，結束程式。
3. 將十進位轉成二進位。
4. 同時計算二進位中 `1` 的數量。
5. 按照題目格式輸出。

**Time Complexity：** `O(log n)`

**Space Complexity：** `O(log n)`
