# UVa 11450 - Wedding Shopping

## 解題思路

這題要在預算 `M` 內購買衣服。

有 `C` 種衣服類別，每一類都必須買 **剛好一件**。

每一類會提供多個價格選項，目標是在不超過預算的情況下，讓總花費最大。

如果無法每一類都買一件，就輸出：

```text
no solution
```

這題可以使用 **Dynamic Programming（DP）** 解決。

---

## 關鍵邏輯

### 1. DP 狀態

使用：

```cpp
dp[i][money]
```

表示：

```text
買完前 i 類衣服後，是否可以花掉 money 元
```

如果可以，值就是 `true`。

---

### 2. 狀態轉移

對於每一類衣服，嘗試每一個價格。

如果之前可以花掉 `spent` 元，現在買一件價格 `price` 的衣服後：

```text
spent + price <= M
```

則：

```cpp
dp[i + 1][spent + price] = true;
```

---

### 3. 找最大花費

處理完所有類別後，從預算 `M` 往下找：

```cpp
for (int money = M; money >= 0; money--)
```

第一個可行的金額就是答案。

---

## 範例推導

假設：

```text
M = 20
C = 3
```

衣服價格：

```text
類別 1：6 8
類別 2：5 10
類別 3：4 7
```

可能組合：

```text
6 + 5 + 4 = 15
6 + 5 + 7 = 18
6 + 10 + 4 = 20
8 + 5 + 7 = 20
```

最大不超過 20 的花費是：

```text
20
```

所以輸出：

```text
20
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 每一類衣服都必須買一件。
2. 不能少買某一類。
3. 不能超過預算 `M`。
4. 若沒有任何合法組合，輸出：

```text
no solution
```

5. 使用 DP 比暴力枚舉所有組合穩定。

---

## 總結（Takeaway）

這題是典型的預算型 DP 題。

解題流程如下：

1. 讀入預算與衣服類別數。
2. 對每一類衣服，嘗試所有價格。
3. 使用 DP 紀錄目前可達成的花費。
4. 最後從預算往下找最大可行值。
5. 若找不到，輸出 `no solution`。

**Time Complexity：** `O(C × M × K)`

**Space Complexity：** `O(C × M)`
