# UVa 679 - Dropping Balls

## 解題思路

這題是在模擬球掉進一棵完全二元樹。

每個節點都有一個開關，球經過時會依照開關狀態決定往左或往右，然後開關會反轉。

但如果真的一顆一顆球去模擬，當球數很多時會太慢。

因此可以觀察規律：

- 奇數編號的球會往左子樹。
- 偶數編號的球會往右子樹。

每往下一層，就根據目前球的編號更新它在子樹中的相對編號。

---

## 關鍵邏輯

### 1. 從根節點開始

根節點編號是：

```text
1
```

若往左子節點：

```text
node = node * 2
```

若往右子節點：

```text
node = node * 2 + 1
```

---

### 2. 根據球的編號決定方向

如果目前 `ball` 是奇數：

```text
往左
```

並更新：

```cpp
ball = (ball + 1) / 2;
```

如果目前 `ball` 是偶數：

```text
往右
```

並更新：

```cpp
ball = ball / 2;
```

---

### 3. 重複 D - 1 層

如果深度是 `D`，球會從根節點一路走到第 `D` 層。

因此只需要走：

```text
D - 1
```

次。

---

## 範例推導

假設：

```text
D = 4
I = 2
```

從根節點開始：

```text
node = 1
ball = 2
```

第一層：

```text
ball 是偶數 → 往右
node = 3
ball = 1
```

第二層：

```text
ball 是奇數 → 往左
node = 6
ball = 1
```

第三層：

```text
ball 是奇數 → 往左
node = 12
```

所以答案是：

```text
12
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不要真的模擬所有球。
2. 只需要追蹤第 `I` 顆球的路徑。
3. 深度為 `D` 時，只走 `D - 1` 次。
4. 左子節點是 `node * 2`。
5. 右子節點是 `node * 2 + 1`。

---

## 總結（Takeaway）

這題是完全二元樹規律題。

解題流程如下：

1. 從節點 `1` 開始。
2. 根據球的編號奇偶決定往左或往右。
3. 每往下一層更新節點編號。
4. 重複 `D - 1` 次。
5. 輸出最後停留的節點。

**Time Complexity：** `O(D)`

**Space Complexity：** `O(1)`
