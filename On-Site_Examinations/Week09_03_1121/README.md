# UVa 1121 - Subsequence

## 解題思路

這題要求找出一個**連續子序列（Subsequence）**，使其總和大於等於 `S`，並輸出其中**最短的長度**。

如果不存在符合條件的子序列，就輸出：

```text
0
```

由於所有數字都是正整數，因此可以使用 **Sliding Window（滑動視窗）** 來解決，不需要暴力列舉所有區間。

---

## 關鍵邏輯

### 1. 使用兩個指標

利用：

```text
left
right
```

代表目前視窗：

```text
[left ... right]
```

並維護目前區間總和：

```cpp
sum
```

---

### 2. 擴大視窗

每讀入一個新數字：

```cpp
sum += a[right];
```

讓總和逐漸增加。

---

### 3. 縮小視窗

只要：

```text
sum >= S
```

代表目前區間已經符合條件。

此時更新答案：

```cpp
answer = min(answer, right - left + 1);
```

然後嘗試把左端移動：

```cpp
sum -= a[left];
left++;
```

看看能不能找到更短的區間。

---

## 範例推導

假設：

```text
S = 15

2 3 1 2 4 3
```

視窗變化：

```text
2 3 1 2 4 3

總和：

2
5
6
8
12
15
```

第一次達到：

```text
2 3 1 2 4 3
```

長度：

```text
6
```

接著縮小：

```text
3 1 2 4 3
```

總和：

```text
13
```

不符合，再往右。

最後找到：

```text
4 3
```

總和：

```text
7
```

前一步：

```text
2 4 3
```

長度：

```text
2
```

答案就是：

```text
2
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不要使用兩層迴圈暴力搜尋，會 TLE。
2. Sliding Window 只適用於全部都是正整數。
3. 每次 `sum >= S` 時，要持續縮小左端。
4. 若沒有符合條件的區間，要輸出：

```text
0
```

---

## 總結（Takeaway）

這題是 Sliding Window 的經典題。

解題流程如下：

1. 建立左右指標。
2. 右指標持續向右擴張。
3. 當總和達到 `S` 時，不斷縮小左端。
4. 更新最短長度。
5. 最後輸出答案。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
