# UVa 10062 - Tell me the frequencies

## 解題思路

這題要統計每一行文字中，每個 ASCII 字元出現的次數。

與一般字母統計不同的是，題目要求：

- 所有 ASCII 字元都要統計（包含空白、數字、符號等）。
- 依照 **出現次數由小到大排序**。
- 如果出現次數相同，則依照 **ASCII 值由大到小排序**。

因此，我們需要先統計每個字元的出現次數，再依照題目要求排序後輸出。

---

## 關鍵邏輯

### 1. 統計每個 ASCII 字元

可以使用大小為 128 的陣列：

```cpp
int freq[128] = {0};
```

掃描整行文字：

```cpp
freq[ch]++;
```

即可完成統計。

---

### 2. 建立排序資料

將所有出現過的字元存入：

```cpp
vector<pair<int,int>>
```

其中：

```text
(first)  ASCII 值
(second) 出現次數
```

方便之後排序。

---

### 3. 排序規則

排序依照：

1. 出現次數由小到大。
2. 若次數相同，ASCII 值由大到小。

例如：

```text
A 2
a 2
```

因為：

```text
'A' = 65
'a' = 97
```

所以輸出順序會是：

```text
97 2
65 2
```

---

## 範例推導

假設輸入：

```text
AAAbb
```

ASCII：

```text
A → 65
b → 98
```

統計：

```text
65 出現 3 次
98 出現 2 次
```

排序後：

```text
98 2
65 3
```

---

## 語言差異與踩坑紀錄

這題最容易錯的地方：

1. 要使用 `getline()`，因為可能有空白。
2. 每一行都是一組新的測資。
3. 不同測資之間要輸出空白行。
4. 排序規則不是 ASCII 由小到大，而是：
   - 次數小的先
   - 次數相同時，ASCII 大的先

---

## 總結（Takeaway）

這題主要考驗：

1. ASCII 字元統計。
2. 自訂排序規則。
3. `getline()` 的使用。

解題流程如下：

1. 讀入一整行文字。
2. 統計每個 ASCII 字元出現次數。
3. 將出現過的字元存入 vector。
4. 按照題目要求排序。
5. 依序輸出 ASCII 值與出現次數。

**Time Complexity：** `O(n + 128 log 128)`

**Space Complexity：** `O(128)`
