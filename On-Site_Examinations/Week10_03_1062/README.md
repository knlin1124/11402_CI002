# UVa 1062 - Containers

## 解題思路

這題要將一串英文字母依序放入多個容器（Stack）中。

放置規則如下：

- 每個容器都是一個 Stack（後進先出）。
- 若目前字母小於或等於某個容器最上面的字母，就可以放進該容器。
- 如果沒有任何容器可以放，就需要建立新的容器。

最後輸出最少需要幾個容器。

---

## 關鍵邏輯

### 1. 依序處理每個字母

每讀到一個字母，就依序檢查目前所有容器。

假設目前字母是：

```text
C
```

若某個容器頂端為：

```text
D
```

因為：

```text
C <= D
```

所以可以直接放進去。

---

### 2. 優先放進第一個可放的容器

找到第一個符合：

```cpp
top >= current
```

的容器，就直接更新容器頂端：

```cpp
top = current;
```

不需要真的使用 Stack。

因為只有最上面的字母會影響後續放置。

---

### 3. 找不到就建立新容器

若所有容器都不能放：

```text
A
B
C
```

目前字母：

```text
Z
```

就建立新的容器。

---

## 範例推導

假設輸入：

```text
ABCD
```

放置過程：

```text
A
AB
ABC
ABCD
```

每個字母都比前一個大，因此需要：

```text
4 個容器
```

---

假設輸入：

```text
DCBA
```

放置：

```text
D
C
B
A
```

全部都可以放到同一個容器。

答案：

```text
1
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不需要真的建立很多 Stack。
2. 只需要記錄每個容器最上面的字母即可。
3. 一定要放進第一個符合條件的容器。
4. 輸入：

```text
end
```

代表結束。

---

## 總結（Takeaway）

這題是 Greedy（貪心法）的經典題。

解題流程如下：

1. 建立一個陣列記錄每個容器頂端字母。
2. 每個字母依序尋找第一個可以放的容器。
3. 若找到，就更新容器頂端。
4. 若找不到，就新增一個容器。
5. 最後輸出容器數量。

**Time Complexity：** `O(n²)`

**Space Complexity：** `O(n)`
