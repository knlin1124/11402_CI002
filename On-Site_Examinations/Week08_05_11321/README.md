# UVa 11321 - Sort! Sort!! and Sort!!!

## 解題思路

這題要依照特殊規則排序一組整數。

輸入會給：

```text
N M
```

其中：

- `N` 是數字個數
- `M` 是取餘數的除數

排序時不是直接照數字大小，而是要依照題目指定規則排序。

---

## 關鍵邏輯

### 1. 先比較 `n % M`

兩個數字先比較它們對 `M` 取餘數的結果。

餘數比較小的排前面：

```cpp
a % M < b % M
```

---

### 2. 如果餘數相同，奇數排在偶數前面

若兩個數字的餘數相同：

- 奇數排前面
- 偶數排後面

---

### 3. 如果都是奇數，數字大的排前面

例如：

```text
7 5 3
```

奇數要由大到小排序。

---

### 4. 如果都是偶數，數字小的排前面

例如：

```text
2 4 6
```

偶數要由小到大排序。

---

## 範例推導

假設：

```text
M = 3
numbers = 1 2 3 4 5
```

先看餘數：

```text
1 % 3 = 1
2 % 3 = 2
3 % 3 = 0
4 % 3 = 1
5 % 3 = 2
```

先依照餘數排序：

```text
餘數 0：3
餘數 1：1 4
餘數 2：2 5
```

同餘數時：

- 奇數排偶數前
- 奇數由大到小
- 偶數由小到大

所以最後會依照規則排序。

---

## 語言差異與踩坑紀錄

這題最容易錯的地方是排序規則。

必須照以下順序判斷：

1. `a % M` 小的先。
2. 餘數相同時，奇數排在偶數前。
3. 都是奇數時，數字大的先。
4. 都是偶數時，數字小的先。

另外要注意：

- 輸入結束條件是：

```text
0 0
```

- 每組測資都要先輸出原本的：

```text
N M
```

- 最後也要輸出：

```text
0 0
```

---

## 總結（Takeaway）

這題是自訂排序規則的經典題。

解題流程如下：

1. 讀入 `N` 和 `M`。
2. 如果是 `0 0`，輸出 `0 0` 後結束。
3. 讀入所有數字。
4. 使用 `sort()` 搭配自訂比較函式。
5. 按照題目要求輸出排序後結果。

**Time Complexity：** `O(N log N)`

**Space Complexity：** `O(N)`
