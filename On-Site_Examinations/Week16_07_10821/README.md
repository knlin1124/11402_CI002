# UVa 10821 - Constructing BST

## 解題思路

這題要根據給定的：

```text
n：節點數量
h：樹高限制
```

建構出一棵 Binary Search Tree（BST）。

節點可以使用：

```text
1 ~ n
```

作為數值。

BST 必須符合：

```text
左子樹 < Root < 右子樹
```

題目要求輸出一種可行的插入順序，使得最後形成的 BST 高度不超過 `h`。

如果無法建構，就輸出：

```text
Impossible.
```

---

## 關鍵邏輯

### 1. 判斷是否可行

高度為 `h` 的二元樹最多可以放：

```text
2^h - 1
```

個節點。

如果：

```text
n > 2^h - 1
```

代表高度不夠，一定無法放下所有節點。

---

### 2. 選擇 Root

每次處理一段區間：

```text
[left, right]
```

從中選一個 `root`。

左邊的數字會成為左子樹：

```text
[left, root - 1]
```

右邊的數字會成為右子樹：

```text
[root + 1, right]
```

選擇 root 時，要確保左右子樹都能在剩下高度中放得下。

---

### 3. 遞迴建構

選好 root 後：

```text
先輸出 root
再建左子樹
再建右子樹
```

這樣輸出的順序就是 BST 的插入順序。

---

## 範例推導

假設：

```text
n = 5
h = 3
```

高度 3 最多可放：

```text
2^3 - 1 = 7
```

所以可行。

可以選：

```text
3
```

作為 root。

左子樹：

```text
1 2
```

右子樹：

```text
4 5
```

可能輸出：

```text
3 1 2 4 5
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 要先判斷是否可能建構。
2. 高度 `h` 最多可放 `2^h - 1` 個節點。
3. 輸出的是插入順序，不是 inorder。
4. 無法建構時輸出：

```text
Impossible.
```

5. 輸出格式要有：

```text
Case X:
```

---

## 總結（Takeaway）

這題是 BST 建構題。

解題流程如下：

1. 讀入 `n` 和 `h`。
2. 判斷高度是否足夠放下所有節點。
3. 若不行，輸出 `Impossible.`。
4. 若可行，遞迴選 root。
5. 輸出一組符合條件的插入順序。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
