# UVa 11286 - Conformity

## 解題思路

這題要統計大學生選課的情況。

每位學生都選了 **5 門課**，課程編號的順序沒有意義，因此：

```text
100 101 102 103 104
```

與

```text
104 103 102 101 100
```

代表的是**同一種選課組合**。

題目要求找出：

> **最受歡迎的選課組合共有多少位學生。**

如果有多個組合並列最受歡迎，就要把所有這些組合的人數加總。

---

## 關鍵邏輯

### 1. 每位學生先排序課程

由於課程順序沒有意義，因此先排序：

```cpp
sort(course.begin(), course.end());
```

例如：

```text
104 102 101 103 100
```

排序後：

```text
100 101 102 103 104
```

這樣所有相同組合都會變成相同表示方式。

---

### 2. 使用 map 統計

可以使用：

```cpp
map<vector<int>, int> cnt;
```

或：

```cpp
map<string, int>
```

統計每種選課組合出現次數。

---

### 3. 找最大出現次數

假設統計結果：

```text
A → 20

B → 18

C → 20

D → 5
```

最大值：

```text
20
```

但是：

```text
A

C
```

都同樣最多。

答案不是：

```text
20
```

而是：

```text
20 + 20 = 40
```

---

## 範例推導

假設：

```text
學生1

100 101 102 103 104
```

學生2：

```text
104 103 102 101 100
```

排序後都變成：

```text
100 101 102 103 104
```

因此：

```text
同一組
```

次數：

```text
2
```

---

假設：

```text
組合A：

3人

組合B：

3人

組合C：

2人
```

最大：

```text
3
```

答案：

```text
3 + 3 = 6
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 課程一定要排序。
2. 課程順序不重要。
3. 並列第一名的人數要全部加總。
4. 使用 `map` 統計最方便。
5. 輸入：

```text
0
```

代表結束。

---

## 總結（Takeaway）

這題主要練習 Map 統計。

解題流程如下：

1. 每位學生先排序五門課。
2. 使用 `map` 統計每種課程組合。
3. 找出最高出現次數。
4. 將所有最高次數的組合人數加總。
5. 輸出答案。

**Time Complexity：** `O(n log n)`

**Space Complexity：** `O(n)`
