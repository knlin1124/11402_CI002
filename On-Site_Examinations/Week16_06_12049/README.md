# UVa 12049 - Just Prune The List

## 解題思路

這題給兩個整數串列，要求最少刪除多少個數字，才能讓兩個串列完全相同。

每次只能刪除元素，不能修改或新增。

因此，只需要統計每個數字在兩個串列中出現的次數，再比較差異即可。

最適合使用 **Map（映射）** 統計每個數字的出現次數。

---

## 關鍵邏輯

### 1. 統計第一個串列

建立：

```cpp
map<int, int> cnt;
```

讀入第一個串列：

```cpp
cnt[x]++;
```

表示數字 `x` 出現一次。

---

### 2. 處理第二個串列

讀入第二個串列時：

如果：

```cpp
cnt[x] > 0
```

代表第一個串列有相同的數字，可以互相抵消：

```cpp
cnt[x]--;
```

否則：

代表第二個串列多出一個數字，需要刪除：

```cpp
answer++;
```

---

### 3. 計算第一個串列剩餘數字

最後 `map` 中仍然大於 0 的數字，就是第一個串列多出的元素。

全部加總即可。

---

## 範例推導

假設：

第一個串列：

```text
1 2 2 3
```

第二個串列：

```text
2 3 4
```

統計：

```text
1 → 1

2 → 2

3 → 1
```

讀第二個串列：

```text
2

↓

2 → 1
```

```text
3

↓

3 → 0
```

```text
4

↓

不存在

答案 +1
```

最後剩下：

```text
1 → 1

2 → 1
```

還需要刪：

```text
2 個
```

總共：

```text
3
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不需要排序。
2. 使用 `map` 統計即可。
3. 相同數字可以互相抵消。
4. 最後要把第一個串列剩下的數量也加進答案。
5. 每組測資都要重新建立 `map`。

---

## 總結（Takeaway）

這題是 Map 計數的經典題。

解題流程如下：

1. 建立 `map<int,int>`。
2. 統計第一個串列所有數字。
3. 掃描第二個串列並互相抵消。
4. 第二個串列多出的數字直接計入答案。
5. 最後再把 `map` 中剩餘的數量加總。

**Time Complexity：** `O((N + M) log N)`

**Space Complexity：** `O(N)`
