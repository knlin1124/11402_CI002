# UVa 11157 - Dynamic Frog

## 解題思路

這題有一隻青蛙要過河。

河中有兩種石頭：

- **B（Big Stone）**：大石頭，可以踩很多次。
- **S（Small Stone）**：小石頭，只能踩一次，踩完就會消失。

目標是：

> **求青蛙過河再回來時，所需的最小最大跳躍距離（Minimize the Maximum Jump）。**

這題使用 **Greedy（貪心法）**。

---

## 關鍵邏輯

### 1. 加入起點與終點

起點：

```text
0
```

終點：

```text
D
```

一起加入石頭列表。

例如：

```text
0

B-3

S-6

S-8

B-12

20
```

---

### 2. 去程策略

因為小石頭只能踩一次，所以不能全部都踩。

最佳策略是：

> **去程每隔一顆踩一次。**

例如：

```text
0
3
6
8
12
20
```

去程：

```text
0 → 6 → 12 → 20
```

留下：

```text
3

8
```

給回程使用。

---

### 3. 回程

回程就踩剩下的石頭：

```text
20 → 12 → 8 → 3 → 0
```

---

### 4. 計算最大跳躍

每一次跳躍：

```text
distance = next - current
```

整個過程最大的距離就是答案。

---

## 範例推導

假設：

```text
D = 25
```

石頭：

```text
B-2

S-11

S-14

B-17

S-21
```

加入：

```text
0

2

11

14

17

21

25
```

去程：

```text
0 → 11 → 17 → 25
```

回程：

```text
25 → 21 → 14 → 2 → 0
```

計算所有跳躍距離：

```text
11

6

8

4

7

12

2
```

最大：

```text
12
```

答案就是：

```text
12
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 大石頭可以踩很多次。
2. 小石頭只能踩一次。
3. 起點與終點都要視為大石頭。
4. 去程每隔一顆踩一次即可得到最佳解。
5. 最後輸出：

```text
Case X: answer
```

---

## 總結（Takeaway）

這題是 Greedy 模擬題。

解題流程如下：

1. 加入起點與終點。
2. 將所有石頭依位置排序。
3. 去程每隔一顆踩一次。
4. 回程踩剩下的石頭。
5. 計算所有跳躍中的最大值。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
