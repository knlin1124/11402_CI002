# UVa 514 - Rails

## 解題思路

這題要判斷火車是否能透過一條 **中繼軌道（Stack）**，排列成指定的順序。

火車會依照：

```text
1 2 3 ... N
```

的順序進站。

中間只有一條支線，因此符合 **Stack（後進先出，LIFO）** 的特性。

我們要判斷是否能利用這個 Stack，讓火車依照題目要求的順序出站。

如果可以，輸出：

```text
Yes
```

否則輸出：

```text
No
```

---

## 關鍵邏輯

### 1. 模擬火車進站

火車一定按照：

```text
1
2
3
...
N
```

依序進站。

因此使用：

```cpp
nextTrain
```

代表下一台要進站的火車。

---

### 2. 使用 Stack 當中繼站

每台火車先進 Stack：

```cpp
st.push(nextTrain);
```

接著檢查：

```text
Stack 頂端
==
目前要出站的車
```

如果相同：

```cpp
st.pop();
```

代表成功出站。

一直重複直到不能再出站。

---

### 3. 判斷是否成功

若最後：

```cpp
Stack 為空
```

代表所有火車都依照要求順序離開。

答案就是：

```text
Yes
```

否則：

```text
No
```

---

## 範例推導

假設：

```text
N = 5

目標：

5 4 3 2 1
```

Stack：

```text
push 1
push 2
push 3
push 4
push 5

pop 5
pop 4
pop 3
pop 2
pop 1
```

成功。

輸出：

```text
Yes
```

---

假設：

```text
2 1 5 3 4
```

模擬後：

```text
最後無法得到 4
```

因此：

```text
No
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 每組測資有很多排列。
2. 排列第一個數字是 `0` 時，代表這組結束。
3. 當 `N = 0` 時，整個輸入結束。
4. 每組測資結束要輸出一個空白行。
5. Stack 每次排列都要重新建立。

---

## 總結（Takeaway）

這題是 Stack 模擬的經典題。

解題流程如下：

1. 火車依序進站。
2. 每台火車先進 Stack。
3. 若 Stack 頂端等於目前需要出站的火車，就立刻出站。
4. 重複直到所有火車都處理完成。
5. 若 Stack 為空，代表排列成功。

**Time Complexity：** `O(N)`

**Space Complexity：** `O(N)`
