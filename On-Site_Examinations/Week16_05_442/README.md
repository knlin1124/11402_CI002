# UVa 442 - Matrix Chain Multiplication

## 解題思路

這題給定多個矩陣的大小，以及許多矩陣乘法運算式。

目標是：

> **計算矩陣相乘所需的乘法次數。**

如果矩陣尺寸不符合乘法規則，就輸出：

```text
error
```

由於運算式已經用括號決定好運算順序，因此不用重新找最佳順序，只需要依照括號模擬即可。

最適合使用 **Stack（堆疊）**。

---

## 關鍵邏輯

### 1. 儲存每個矩陣大小

建立：

```cpp
map<char, pair<int,int>>
```

例如：

```text
A

50 × 10
```

表示：

```text
matrix['A'] = {50,10}
```

---

### 2. 遇到矩陣名稱

例如：

```text
A
```

直接放入 Stack。

Stack 裡存的是：

```text
(row, column)
```

---

### 3. 遇到 ')'

代表完成一個乘法。

例如：

```text
(AB)
```

Stack：

```text
A

B
```

先取：

```text
B

A
```

再檢查：

```text
A.col == B.row
```

若：

```text
10 == 10
```

即可相乘。

乘法次數：

```text
A.row × A.col × B.col
```

新的矩陣：

```text
A.row × B.col
```

再放回 Stack。

---

## 範例推導

假設：

```text
A

50×10
```

```text
B

10×20
```

運算：

```text
(AB)
```

乘法次數：

```text
50 × 10 × 20

=

10000
```

新矩陣：

```text
50 × 20
```

---

若：

```text
A

50×10
```

```text
B

30×20
```

因為：

```text
10 ≠ 30
```

所以：

```text
error
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不需要重新排列矩陣順序。
2. 題目的括號就是運算順序。
3. Stack 裡存的是矩陣大小，不是字母。
4. 相乘條件：

```text
A.col == B.row
```

5. 若不符合，直接輸出：

```text
error
```

---

## 總結（Takeaway）

這題是 Stack 模擬的經典題。

解題流程如下：

1. 建立每個矩陣大小。
2. 遇到矩陣放入 Stack。
3. 遇到 ')' 就彈出兩個矩陣。
4. 檢查是否可相乘。
5. 計算乘法次數並放回新的矩陣。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
