# UVa 10020 - Minimal Coverage

## 解題思路

這題會給一個目標區間：

```text
[0, M]
```

以及很多線段：

```text
(left, right)
```

目標是：

> **用最少的線段完全覆蓋 [0, M]。**

如果無法完整覆蓋，就輸出：

```text
0
```

這題是一道非常經典的 **Greedy（貪心法）** 題目。

---

## 關鍵邏輯

### 1. 將所有線段排序

依照：

```text
左端點由小到大
```

排序。

若左端點相同，再依：

```text
右端點由大到小
```

排序。

```cpp
sort(segment.begin(), segment.end());
```

---

### 2. 每次找能延伸最遠的線段

目前已覆蓋到：

```text
current
```

所有：

```text
left <= current
```

的線段都可以選。

從這些線段中挑：

```text
right 最大
```

的那一條。

因為它可以讓覆蓋範圍延伸最遠。

---

### 3. 更新覆蓋範圍

假設目前：

```text
current = 7
```

找到：

```text
(5,12)
```

那麼：

```text
current = 12
```

並把這條線段加入答案。

一直重複直到：

```text
current >= M
```

---

## 範例推導

假設：

```text
M = 10
```

線段：

```text
(-1,3)

(2,7)

(5,10)

(6,15)
```

第一次：

```text
目前覆蓋：

0
```

可以選：

```text
(-1,3)
```

更新：

```text
current = 3
```

第二次：

```text
可以選：

(2,7)
```

更新：

```text
current = 7
```

第三次：

```text
可以選：

(5,10)

(6,15)
```

最佳：

```text
(6,15)
```

更新：

```text
current = 15
```

已經覆蓋：

```text
0~10
```

完成。

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 一定要排序。
2. 每次不是選最短，而是選目前能延伸最遠。
3. 若某一步沒有任何線段可選，就代表失敗。
4. 最後輸出使用的所有線段。
5. 每組測資之間要空一行。

---

## 總結（Takeaway）

這題是區間覆蓋（Interval Cover）的經典 Greedy 題。

解題流程如下：

1. 將所有線段排序。
2. 每次找所有 `left <= current` 的線段。
3. 從中選擇 `right` 最大的線段。
4. 更新覆蓋範圍。
5. 若成功覆蓋到 `M`，輸出答案；否則輸出 `0`。

**Time Complexity：** `O(n log n)`

**Space Complexity：** `O(n)`
