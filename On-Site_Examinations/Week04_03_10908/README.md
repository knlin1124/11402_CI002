# UVa 10908 - Largest Square

## 解題思路

這題要在一個字元矩陣中，對每個查詢點 `(r, c)`，找出以這個點為中心，且四周全部都是相同字元的最大正方形。

正方形的邊長一定是奇數，例如：

```text
1, 3, 5, 7 ...
```

因為它必須以查詢點作為正中心。

---

## 關鍵邏輯

### 1. 從中心點開始向外擴張

先取得中心點的字元：

```cpp
char target = grid[r][c];
```

然後從邊長 `1` 開始，逐層往外擴張。

每擴張一層，就檢查新的外框範圍內是否全部都是 `target`。

---

### 2. 檢查是否超出邊界

假設目前擴張半徑是 `len`，要檢查的範圍是：

```text
r - len 到 r + len
c - len 到 c + len
```

只要超出矩陣範圍，就不能再擴張。

---

### 3. 更新最大邊長

如果目前這一圈全部合法，就更新答案：

```cpp
answer = 2 * len + 1;
```

最後輸出最大的奇數邊長。

---

## 範例推導

假設查詢點中心是 `A`：

```text
AAAAA
AAAAA
AAAAA
AAAAA
AAAAA
```

中心點可以先形成：

```text
1 x 1
```

再檢查：

```text
3 x 3
```

如果都一樣，再檢查：

```text
5 x 5
```

直到遇到不同字元或超出邊界為止。

---

## 語言差異與踩坑紀錄

這題最容易錯的地方是：

1. 查詢座標 `(r, c)` 是以 `0` 為起點。
2. 每組測資要先輸出：

```text
M N Q
```

3. 正方形邊長一定是奇數。
4. 擴張時不能只檢查四個角，要檢查整個新增的外框範圍。
5. 一旦超出邊界，就要停止擴張。

---

## 總結（Takeaway）

這題的重點是中心擴張法：

1. 以查詢點為中心。
2. 記錄中心字元。
3. 一圈一圈向外檢查。
4. 只要遇到不同字元或超出邊界就停止。
5. 輸出目前找到的最大正方形邊長。

本題可以用模擬方式完成，邏輯清楚，比較重要的是邊界判斷。
