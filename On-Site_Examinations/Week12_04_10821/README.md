# UVa 10821 - Constructing BST

## 解題思路

這題要根據給定的：

```text
n：節點數量
h：樹高限制
```

建構一棵 Binary Search Tree（BST）。

題目要求輸出一種可行的節點排列方式，使得用這些節點插入 BST 後，樹的高度符合限制。

如果無法建構，就輸出：

```text
Impossible.
```

---

## 關鍵邏輯

### 1. BST 的節點編號

通常可以使用：

```text
1 ~ n
```

作為 BST 的節點值。

BST 的性質是：

```text
左子樹 < Root < 右子樹
```

因此只要選定一個 Root，左邊的數字放左子樹，右邊的數字放右子樹。

---

### 2. 高度限制

若有 `n` 個節點，高度限制為 `h`。

最少需要的高度大約是平衡樹高度。

最多可以做成一條鏈，使用高度 `n`。

如果：

```text
h > n
```

或高度太小無法容納 `n` 個節點，就不可能。

---

### 3. 遞迴建構

每次建構一段區間：

```text
[left, right]
```

並選一個 Root。

選 Root 時要確保剩下的節點能在剩餘高度內放完。

---

## 範例推導

假設：

```text
n = 5
h = 3
```

可以選：

```text
3
```

作為 Root。

左子樹：

```text
1 2
```

右子樹：

```text
4 5
```

形成高度 3 的 BST。

---

## 語言差異與踩坑紀錄

這題容易錯的地方：

1. 要先判斷是否可能建構。
2. BST 必須符合左小右大的規則。
3. 輸出格式要注意 Case 編號。
4. 無法建構時輸出：

```text
Impossible.
```

5. 遞迴時要控制剩餘高度。

---

## 總結（Takeaway）

這題是 BST 建構題。

解題流程如下：

1. 讀入 `n`、`h`。
2. 判斷是否能在高度 `h` 內放入 `n` 個節點。
3. 若不行，輸出 `Impossible.`。
4. 若可行，遞迴選 Root 並建構左右子樹。
5. 輸出符合條件的插入順序。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
