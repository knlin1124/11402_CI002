# UVa 12356 - Army Buddies

## 解題思路

這題模擬一排士兵。

一開始共有：

```text
1 2 3 4 ... S
```

每次戰鬥都會有一段士兵陣亡：

```text
L ~ R
```

題目要求輸出：

```text
L 左邊最近還活著的人

R 右邊最近還活著的人
```

如果不存在，就輸出：

```text
*
```

如果每次都真的刪除士兵，再去搜尋左右鄰居，效率會太慢。

因此使用 **雙向 Linked List（左右鄰居陣列）**。

---

## 關鍵邏輯

### 1. 建立左右鄰居

建立兩個陣列：

```cpp
left[i]
right[i]
```

代表：

```text
i 左邊最近活著的人

i 右邊最近活著的人
```

初始化：

```text
1

left = *

right = 2
```

```text
2

left = 1

right = 3
```

一直到：

```text
S

left = S-1

right = *
```

---

### 2. 士兵陣亡

若：

```text
L ~ R
```

全部死亡。

更新：

```cpp
right[left[L]] = right[R];
```

以及：

```cpp
left[right[R]] = left[L];
```

即可。

完全不用真的刪除士兵。

---

### 3. 輸出答案

左邊：

```cpp
left[L]
```

若不存在：

```text
*
```

右邊：

```cpp
right[R]
```

若不存在：

```text
*
```

---

## 範例推導

假設：

```text
1 2 3 4 5 6
```

死亡：

```text
2 ~ 4
```

剩下：

```text
1 5 6
```

輸出：

```text
1 5
```

再死亡：

```text
5 ~ 6
```

剩下：

```text
1
```

輸出：

```text
1 *
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不要真的刪除士兵。
2. 使用左右鄰居陣列即可。
3. 更新左右鄰居時要判斷是否存在。
4. 每組測資最後輸出：

```text
-
```

5. 題目直到：

```text
0 0
```

結束。

---

## 總結（Takeaway）

這題是 Linked List 模擬的經典題。

解題流程如下：

1. 建立每位士兵左右鄰居。
2. 每次刪除一段士兵。
3. 更新左右鄰居。
4. 輸出左右仍存活的士兵。
5. 不需要真的刪除任何元素。

**Time Complexity：** `O(B)`

**Space Complexity：** `O(S)`
