# UVa 900 - Brick Wall Patterns

## 解題思路

這題要計算有多少種方法可以鋪滿一面長度為 `n` 的牆。

可以使用兩種磚塊：

- 長度 **1**
- 長度 **2**

每塊磚的高度都相同，因此只需考慮水平方向的排列方式。

題目要求：

> **計算共有多少種不同的鋪法。**

這是一道典型的 **Dynamic Programming（DP）** 題，也就是著名的 **Fibonacci（費波那契數列）** 應用。

---

## 關鍵邏輯

### 1. DP 定義

令：

```cpp
dp[i]
```

表示：

```text
長度為 i 的牆共有多少種鋪法。
```

---

### 2. 狀態轉移

最後一塊磚只有兩種可能：

#### (1) 最後放長度 1 的磚

前面剩：

```text
i - 1
```

種鋪法：

```cpp
dp[i-1]
```

---

#### (2) 最後放長度 2 的磚

前面剩：

```text
i - 2
```

種鋪法：

```cpp
dp[i-2]
```

---

因此：

```cpp
dp[i] = dp[i-1] + dp[i-2];
```

這就是 Fibonacci 遞迴。

---

### 3. 初始條件

長度：

```text
1
```

只有：

```text
1
```

種。

```cpp
dp[1] = 1;
```

---

長度：

```text
2
```

可以：

```text
1+1

2
```

共有：

```text
2
```

種。

```cpp
dp[2] = 2;
```

---

## 範例推導

假設：

```text
n = 5
```

DP：

```text
dp[1] = 1

dp[2] = 2

dp[3] = 3

dp[4] = 5

dp[5] = 8
```

答案：

```text
8
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 這不是普通 Fibonacci，而是：

```text
1

2

3

5

8...
```

起始值不同。

2. 可先預處理所有答案。
3. 題目直到：

```text
0
```

結束。
4. 使用 `long long` 儲存答案較安全。

---

## 總結（Takeaway）

這題是 Fibonacci DP 的經典題。

解題流程如下：

1. 預先計算所有 `dp`。
2. 使用：

```cpp
dp[i] = dp[i-1] + dp[i-2];
```

3. 每次直接輸出答案。

**Time Complexity：**

```text
預處理：O(n)

查詢：O(1)
```

**Space Complexity：** `O(n)`
