# UVa 10252 - Common Permutation

## 解題思路

這題要求找出兩個字串共同擁有的字元，並依照字母順序輸出。

如果某個字元在兩個字串中都出現多次，就輸出共同出現的最小次數。

例如：

```text
String A：pretty
String B：women
```

字元統計：

```text
A：

p → 1
r → 1
e → 1
t → 2
y → 1
```

```text
B：

w → 1
o → 1
m → 1
e → 1
n → 1
```

共同只有：

```text
e
```

因此答案：

```text
e
```

---

## 關鍵邏輯

### 1. 統計每個字母出現次數

因為題目只有：

```text
a ~ z
```

所以建立兩個大小 26 的陣列：

```cpp
cntA[26]
cntB[26]
```

分別紀錄每個字母出現次數。

---

### 2. 找共同出現次數

對於每個字母：

```cpp
min(cntA[i], cntB[i])
```

就是共同擁有的數量。

例如：

```text
A：

a → 5
```

```text
B：

a → 2
```

共同只能輸出：

```text
aa
```

---

### 3. 依照字母順序輸出

最後從：

```text
a → z
```

依序輸出即可。

因此答案天然就是字典序。

---

## 範例推導

假設：

```text
A：

acabc
```

```text
B：

abc
```

統計：

```text
A：

a → 2
b → 1
c → 2
```

```text
B：

a → 1
b → 1
c → 1
```

共同：

```text
a → 1

b → 1

c → 1
```

答案：

```text
abc
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 要使用 `getline()`。
2. 字串可能包含空字串。
3. 相同字母要輸出最小次數。
4. 最後依照 a~z 順序輸出。
5. 題目直到 EOF。

---

## 總結（Takeaway）

這題主要練習字元計數。

解題流程如下：

1. 讀入兩個字串。
2. 分別統計每個字母數量。
3. 對每個字母取最小值。
4. 依照字母順序輸出。

**Time Complexity：** `O(n + m + 26)`

**Space Complexity：** `O(26)`
