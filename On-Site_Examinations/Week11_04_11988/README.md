# UVa 11988 - Broken Keyboard (a.k.a. Beiju Text)

## 解題思路

這題要模擬壞掉的鍵盤輸入。

輸入字串中會出現：

```text
[
]
```

其中：

- `[`：游標移到文字最前面
- `]`：游標移到文字最後面
- 其他字元：依照目前游標位置插入文字

如果直接在字串中間插入，效率會很差，因此可以使用 **list** 來處理。

`list` 支援在中間位置快速插入字元，適合這題。

---

## 關鍵邏輯

### 1. 使用 list 儲存文字

```cpp
list<char> text;
```

並使用 iterator 表示目前游標位置：

```cpp
auto it = text.begin();
```

---

### 2. 遇到 `[` 時

游標移到最前面：

```cpp
it = text.begin();
```

---

### 3. 遇到 `]` 時

游標移到最後面：

```cpp
it = text.end();
```

---

### 4. 遇到一般字元

將字元插入目前游標位置：

```cpp
text.insert(it, ch);
```

---

## 範例推導

假設輸入：

```text
This_is_a_[Beiju]_text
```

處理到 `[` 後，游標移到最前面。

接著輸入：

```text
Beiju
```

會被插入到最前面。

最後結果變成：

```text
BeijuThis_is_a__text
```

---

## 語言差異與踩坑紀錄

這題容易犯的錯誤：

1. 不要使用普通 `string` 在中間插入，可能會太慢。
2. 要使用 `getline()` 讀整行。
3. `[` 是游標移到開頭。
4. `]` 是游標移到結尾。
5. 輸入直到 EOF。

---

## 總結（Takeaway）

這題是資料結構模擬題。

解題流程如下：

1. 使用 `list<char>` 儲存文字。
2. 使用 iterator 代表目前游標位置。
3. 遇到 `[` 移到開頭。
4. 遇到 `]` 移到結尾。
5. 其他字元插入目前游標位置。
6. 最後輸出整個 list。

**Time Complexity：** `O(n)`

**Space Complexity：** `O(n)`
