
# UVa 501 - Black Box

## 解題思路

這題要模擬一個 Black Box。

題目會給兩組資料：

1. `ADD` 序列：依序加入黑盒子的數字。
2. `GET` 序列：表示加入到第幾個數字時，要輸出目前第幾小的數字。

每次執行 `GET` 時，並不是固定輸出最小值，而是輸出目前排序後的第 `k` 小數字。

其中 `k` 會隨著查詢次數增加：

```text
第一次 GET：輸出第 1 小
第二次 GET：輸出第 2 小
第三次 GET：輸出第 3 小
```

---

## 關鍵邏輯

### 1. 使用 multiset 維持排序

因為數字會一直被加入，而且可能有重複值，所以適合使用：

```cpp
multiset<int> box;
```

`multiset` 可以：

- 自動排序
- 保留重複數字
- 支援插入新資料

---

### 2. 使用 iterator 指向目前答案

我們用一個 iterator `it` 指向目前要輸出的第 `k` 小數字。

當新增一個數字時，如果新數字比目前 `it` 指向的值還小，代表目前答案的位置需要往前移。

每次 `GET` 後，輸出 `*it`，再讓 `it++`，準備下一次輸出第 `k+1` 小的數字。

---

## 範例推導

假設加入序列：

```text
3 1 5 2
```

GET 序列：

```text
1 2 4
```

第一次 GET：

```text
加入 3
目前資料：[3]
輸出第 1 小：3
```

第二次 GET：

```text
加入 1
目前資料：[1, 3]
輸出第 2 小：3
```

第三次 GET：

```text
加入 5, 2
目前資料：[1, 2, 3, 5]
輸出第 3 小：3
```

---

## 語言差異與踩坑紀錄

這題容易錯的地方：

1. 數字可能重複，所以不能用 `set`。
2. `GET` 不是輸出最小值，而是輸出第 `k` 小。
3. 每次查詢後，`k` 會自動加一。
4. 多組測資之間要輸出空白行。
5. 要小心 iterator 的移動時機。

---

## 總結（Takeaway）

這題是資料結構與排序維護的題目。

核心想法是：

1. 使用 `multiset` 儲存目前加入的所有數字。
2. 用 iterator 維護目前第 `k` 小的位置。
3. 每次 `GET` 輸出目前答案，再把 iterator 往後移。

**Time Complexity：** `O((M + N) log M)`

**Space Complexity：** `O(M)`
