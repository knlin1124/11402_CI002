# UVa 10341 - Solve It

## 解題思路

這題要解一個方程式：

```text
p * e^(-x) + q * sin(x) + r * cos(x) + s * tan(x) + t * x^2 + u = 0
```

其中 `x` 的範圍固定在：

```text
0 <= x <= 1
```

題目要求如果有解，就輸出到小數點後四位；如果沒有解，就輸出：

```text
No solution
```

---

## 關鍵邏輯

### 1. 觀察函數是否有解

令：

```cpp
f(x) = p * exp(-x) + q * sin(x) + r * cos(x) + s * tan(x) + t * x * x + u
```

因為 `x` 只會在 `[0, 1]` 之間，所以可以先檢查：

```cpp
f(0)
f(1)
```

如果 `f(0)` 和 `f(1)` 同號，代表在這個範圍內沒有穿過 0，通常就沒有解。

---

### 2. 使用 Binary Search 找答案

如果 `f(0)` 和 `f(1)` 異號，就可以使用二分搜尋。

每次取中間值：

```cpp
mid = (left + right) / 2
```

計算：

```cpp
f(mid)
```

如果 `f(mid)` 太大或太小，就縮小搜尋範圍。

重複多次後，就可以得到非常接近答案的 `x`。

---

## 範例推導

假設：

```text
f(0) > 0
f(1) < 0
```

代表答案在 `0` 和 `1` 之間。

第一次取：

```text
mid = 0.5
```

如果：

```text
f(0.5) > 0
```

代表答案在右半邊：

```text
0.5 ~ 1
```

繼續二分直到誤差足夠小。

---

## 語言差異與踩坑紀錄

這題容易錯的地方：

1. 要使用 `double`。
2. 要包含 `<cmath>`。
3. 輸出要固定小數點後四位：

```cpp
cout << fixed << setprecision(4);
```

4. 如果沒有解，要輸出：

```text
No solution
```

5. 二分搜尋建議跑固定次數，例如 100 次，避免精度問題。

---

## 總結（Takeaway）

這題是浮點數二分搜尋的經典題。

解題流程如下：

1. 定義函數 `f(x)`。
2. 檢查 `f(0)` 和 `f(1)` 是否有機會產生解。
3. 若無解，輸出 `No solution`。
4. 若有解，使用 Binary Search 在 `[0,1]` 中找根。
5. 輸出答案到小數點後四位。

**Time Complexity：** `O(1)`

**Space Complexity：** `O(1)`
