# UVa 10008 - What's Cryptanalysis?

## 解題思路

這題要統計多行文字中，每個英文字母出現的次數。

不管是大寫還是小寫，都要視為同一個字母。例如：

```text
A 和 a 都算 A
```

最後依照出現次數由大到小排序，如果出現次數相同，則依照字母順序由小到大排序。

---

## 關鍵邏輯

### 1. 統計字母出現次數

先讀入測資行數 `n`，接著讀取 `n` 行文字。

對每個字元判斷是否為英文字母：

```cpp
isalpha(ch)
```

如果是字母，就轉成大寫：

```cpp
toupper(ch)
```

再統計次數。

---

### 2. 排序規則

輸出時要依照：

1. 出現次數多的排前面。
2. 次數相同時，字母較小的排前面。

例如：

```text
A 5
B 5
C 3
```

因為 `A` 和 `B` 次數相同，所以 `A` 先輸出。

---

## 範例推導

假設文字中出現：

```text
A a B b b C
```

轉成大寫後：

```text
A A B B B C
```

統計結果：

```text
A 2
B 3
C 1
```

排序後輸出：

```text
B 3
A 2
C 1
```

---

## 語言差異與踩坑紀錄

這題最容易錯的地方是輸入讀取。

因為第一行是數字，後面是整行文字，所以在讀完 `n` 之後，要先使用：

```cpp
cin.ignore();
```

把換行符號清掉，否則第一個 `getline()` 可能會讀到空行。

另外也要注意：

1. 只統計英文字母。
2. 不分大小寫，全部轉成大寫。
3. 輸出格式是：

```text
字母 次數
```

---

## 總結（Takeaway）

這題是字元處理與排序的基本題。

解題步驟如下：

1. 讀入多行文字。
2. 判斷字元是否為英文字母。
3. 將字母轉成大寫。
4. 統計每個字母出現次數。
5. 按照題目要求排序後輸出。

**Time Complexity：** `O(n × m + 26 log 26)`

**Space Complexity：** `O(26)`
